如何在博客中提供软件安装包或文件下载？

由于你使用的是 GitHub Pages 托管博客，GitHub 对代码仓库有严格的大小限制（单个文件最大不能超过 100MB，整个仓库建议不要超过 1GB）。因此，你需要根据文件的大小来决定不同的上传方式：

情况一：小文件（小于 20MB）

比如一些小工具、PDF 文档、代码压缩包。

  1. 放置文件：你可以直接把文件复制到博客根目录的 source/downloads/ 文件夹下。
    （例如：把 v2rayN-Core.zip 放到 F:\MyBlog\myblogs\source\downloads\v2rayN-Core.zip）

情况二：大文件 / 软件安装包（大于 20MB）

比如几百兆的安装程序、游戏包、大型数据集。坚决不能直接放进博客源码里！ 否则会导致你的 GitHub 仓库爆炸，最后无法部署博客。

推荐的做法（使用云盘/网盘）：

  1. 上传到云盘：
    • 推荐使用：蓝奏云（对小软件极其友好，不限速，单文件限制 100M）
    • 或者使用：阿里云盘、百度网盘、123云盘、Google Drive、OneDrive 等。
  2. 获取分享链接：在网盘里生成公开分享链接（如果有提取码也要记下）。

开发者进阶方案：GitHub Releases
如果你要分享的是开源软件或者脚本，你可以在 GitHub 新建一个专门的空仓库，然后在那个仓库里发布 Release，把安装包上传到 Release 中。然后把那个 Release 的下载链接写在博客里。这种方式完全免费且显得非常专业。
